Some types in Python can be modified after creation (mutable), while others cannot (immutable). This distinction is crucial for understanding how Python works. Key to this is understanding the difference between reassignment and mutation.

Mutation

Mutation changes the contents of an existing object and can only be performed on mutable types.

my_list = [1, 2, 3] print(my_list) my_list[0] = 10 # Change the contents of the existing list print(my_list) It is not possible to change the contents of an immutable object. For example, you cannot change a character in a string: message="hello" message[0] = "H"

Reassignment

Reassignment of a variable changes the variable to refer to a different object (this can be done for immutable or mutable types):

x = 5 x = 10 print(x) y = [1, 2, 3] y = [4, 5, 6] # Create a new list object print(y)

Multiple References

When multiple variables refer to the same mutable object, mutation of that object can be seen from any variable referencing that object:

x = "hello" list1 = [1, 2, 3] list2 = list1 list2[0] = 10 print(list1) Multiple variables can also refer to the same immutable object, but since it cannot be changed, we can only reassign the variable, which does not affect the object the other variables reference: x = "hello" y = x y = "goodbye" # This creates a new string object, x still references "hello" print(x) print(y)
