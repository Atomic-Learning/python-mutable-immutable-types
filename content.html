Some types in Python can be modified after creation (mutable), while others cannot (immutable). This distinction is crucial for understanding how Python works.

Immutable Types (Cannot Be Changed)

Immutable types: int, float, complex, bool, str, tuple (and a few others)

Once created, you cannot modify them. To "change" an immutable value, you create a new object:

x = 5 x = x + 1 print(x) message = "hello" message = message + " world" print(message)

Note that we're creating a new object each time we "change" an immutable value.

Mutable Types (Can Be Changed)

Mutable types: list, dict, and many others

Mutable types can be modified in place. The variable still refers to the same object, but its contents change:

my_list = [1, 2, 3] my_list[0] = 10 print(my_list) my_dict = {"name": "Alice"} my_dict["name"] = "Bob" print(my_dict)

Key Difference: Reassignment vs. Mutation

Reassignment: Pointing the variable to a new object (works for any type)

x = 5 x = 10

Mutation: Changing the contents of an existing object (only mutable types)

my_list = [1, 2, 3]
my_list[0] = 10  # Change the contents of the existing list

Multiple References

When multiple variables refer to the same mutable object, changes affect all of them:

x = "hello" y = x y = "goodbye" # This creates a new string object, x still points to "hello" print(x) print(y) list1 = [1, 2, 3] list2 = list1 list2[0] = 10 print(list1)

New Mutable Objects

It is still possible to create a new mutable object by assigning a new list (or other mutable type) to a variable:

list1 = [1, 2, 3] list2 = list1 list1 = [4, 5, 6] # list1 now points to a new list, list2 still points to the original print(list1) print(list2)
